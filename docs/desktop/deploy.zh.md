# 部署、打包与发布

本页仅面向桌面程序的运行、打包与发布，不包含服务端相关内容。

## 运行要求

- 需要完整的运行环境与依赖组件，建议使用项目提供的依赖清单进行安装
- 若需要生成文档预览，请在本机提前准备好排版环境与相关字体

## 从源码运行

开发态运行时，建议在隔离环境中安装依赖后启动主程序。遇到问题可先查看运行日志与错误提示。

## 打包发布

项目内已包含打包配置与脚本，可用于生成可分发的桌面程序。打包时建议重点检查：

- 资源文件是否完整（图标、离线文档等）
- 临时目录是否可写（用于生成中间文件与预览）
- 目标机器的字体与排版环境是否满足需求

## 常见问题排查

- 启动失败：优先检查依赖是否安装完整，以及运行环境是否一致
- 文档预览失败：优先检查排版环境是否可用，并查看运行日志中的错误信息
